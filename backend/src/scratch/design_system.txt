AtomQuest 1.0
Professional Design System
Goal Setting & Tracking Portal  ·  Hackathon 1.0
Context & Design Rationale
This is an Enterprise HR Goal-Tracking SaaS Portal — not a startup landing page. The users are employees, managers, and HR admins operating within structured workflows. The design must communicate trust, clarity, authority, and structure while avoiding the cold sterility of legacy enterprise tools.
Reference benchmarks: Linear, Notion, Rippling — structured but warm. Every decision below is grounded in that positioning.

4.  Usage Rules & Anti-Patterns
■  Typography Rules
Instrument Serif is display-only — never below 40px, never for UI labels
Syne is numbers-only — never use it for prose or UI copy
Chivo Mono is code/data only — audit trail entries, goal IDs, timestamps, hex codes
Never mix Bricolage and Geist in the same UI element
Body text minimum 16px (Geist 400) — never go below for primary content
■  Color Rules
Components reference CSS tokens only — never hardcode hex values in component code
Semantic colors (success/warning/danger/info) are for workflow states only — not decoration
Accent teal is never used as a primary action color — that role belongs to primary blue
Both the foreground and -bg surface token must be used together for semantic states
All text must meet WCAG AA contrast (4.5:1 body, 3:1 large text) against its background surface
